# Loomgen HTML Element Properties and Atomic Rule Dispatch Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Complete #607 by generating HTML element classification and wiring a real, compile-once HostGuard close-boundary dispatch with scoped parser-stack ownership.

**Architecture:** Static HTML tag properties remain generated from `#loom.term` metadata. The grammar interpreter keeps its existing non-transactional `Unit` expression semantics and adds one native-only atomic gate, `try_parse_rule(name) -> Bool`, restricted to named rules whose top-level expression is `Choice`. The gate evaluates ordered arm predicates first and executes the first matching body once. HTML parser frames own their stack entries and unwind them on every return path.

**Tech Stack:** MoonBit, `@grammar` IR/interpreter, `@core.ParserContext`, loomgen-generated MoonBit artifacts, native-target focused tests.
**Status:** Complete

## Global Constraints

- Preserve `OpenTag(String)` and `CloseTag(String)` payloads and source-span fidelity.
- Do not add a second HTML element taxonomy or handwritten membership table.
- Do not propagate a general parse-result/backtracking value through `Expr`.
- `try_parse_rule(false)` guarantees no interpreter-owned body, cursor, CST, or diagnostic mutation; arbitrary `HostGuard` closures must be pure queries by implementation contract.
- Native dispatch targets are top-level `Choice` rules; compile-time metadata validates declared targets and runtime dispatch rejects undeclared calls.
- Every `parse_element` frame unwinds the stack entry it owns on **all** return paths: matching close, mismatched close, EOF (unclosed), depth-limit recovery, and error-limit recovery.
- Use `rtk` for every shell command; run `moon check` after every code edit.
- Keep generated artifacts regenerated by loomgen; do not hand-author generated files.

## Task 3: Implement `try_parse_rule` without general backtracking

**Files:**
- Modify: `loom/grammar/interpreter.mbt`
- Modify: `loom/grammar/interpreter_test.mbt`

**Interfaces:**
- `NativeRule[T, K]` changes from `(ParserContext, (RuleName) -> Unit) -> Unit` to `(ParserContext, (RuleName) -> Bool) -> Unit`.
- `interpret_compiled` constructs a `try_parse_rule` closure that captures `ctx`, `grammar`, `natives`, `guards`.
- The `try_parse_rule` dispatcher:
  ```text
  resolve name → slot in grammar.names
  if name is not a declared dependency of any native → emit diagnostic, return false
  if grammar.rule(slot) is not Choice → emit diagnostic, return false
  scan Choice arms in authored order
  call pred_matches_ctx on each arm.starts
  on first match: run_expr(arm.body) exactly once → return true
  when no arm matches: return false without executing any body
  ```
  The false path must not call `ctx.error`, `ctx.emit_*`, `ctx.skip_*`, or any cursor-mutating helper. It does not attempt rollback.

- `run_expr` remains `-> Unit`; no `RuleResult` is introduced.

- [x] **Step 3: Implement the `try_parse_rule` helper function**

Add a new private function to `interpreter.mbt`:

```moonbit
fn try_parse_rule[T, K](
  ctx, grammar, natives, guards,
  caller_name : RuleName,
  target_name : RuleName,
) -> Bool {
  // 1. Resolve target_name to a slot
  match grammar.names.search(target_name) {
    None => {
      ctx.error("grammar: try_parse_rule(\"" + target_name + "\") is not a grammar rule")
      ctx.emit_error_placeholder()
      return false
    }
    Some(slot) => ()
  }
  // 2. Check the target is a declared dependency of caller_name
  match grammar.native_rule_refs.get(caller_name) {
    None => {
      ctx.error("grammar: native \"" + caller_name + "\" has no declared dispatch dependencies")
      ctx.emit_error_placeholder()
      return false
    }
    Some(allowed) =>
      if !allowed.contains(target_name) {
        ctx.error("grammar: native \"" + caller_name + "\" cannot dispatch to \"" + target_name + "\" — not in declared dependencies")
        ctx.emit_error_placeholder()
        return false
      }
  }
  // 3. Require top-level Choice
  match grammar.rule(slot) {
    CompiledExpr::Choice(alts) => {
      let token = ctx.peek()
      for alt in alts {
        if pred_matches_ctx(alt.starts, token, ctx, guards) {
          run_expr(ctx, grammar, natives, guards, alt.body)
          return true
        }
      }
      return false
    }
    _ => {
      ctx.error("grammar: try_parse_rule(\"" + target_name + "\") target is not a Choice rule")
      ctx.emit_error_placeholder()
      return false
    }
  }
}
```

Note: checking `grammar.names.search` on each call is O(n). As a follow-up optimization (outside #607), the slot could be pre-resolved and stored in `CompiledGrammar` if profiling shows it matters. The current `names` array is typically < 20 entries for HTML.

```moonbit
fn parse_element(ctx, tag_stack, try_parse_rule, depth~) -> Unit {
  // depth and error guards (no tag on stack yet — early return is safe)
  if depth > MAX_DEPTH { ...; return }
  if ctx.too_many_errors(MAX_ERRORS) { ...; return }

  match ctx.peek() {
    @token.OpenTag(name) => {
      let tag_name = name
      let kind = @syntax.classify_element(tag_name)
      let is_void = match kind { ... }
      let is_raw = match kind { ... }
      ctx.start_node(@syntax.ElementNode)
      ctx.emit_token(@syntax.OpenTagToken)

      if is_void {
        ctx.finish_node()
        return
      }

      // RECORD base BEFORE pushing
      let base = tag_stack.length()
      let normalized_name = ascii_lower_tag_name(tag_name)
      tag_stack.push(normalized_name)

      if is_raw {
        match ctx.peek() {
          @token.RawText => { ... }
          _ => ()
        }
        finish_element(ctx, tag_stack, try_parse_rule, tag_name)
        // UNWIND before returning
        while tag_stack.length() > base {
          ignore(tag_stack.pop())
        }
        ctx.finish_node()
        return
      }

      parse_content(ctx, tag_stack, try_parse_rule, tag_name, depth=depth + 1)
      // parse_content returns on both successful close AND mismatched close.
      // UNWIND in both cases.
      while tag_stack.length() > base {
        ignore(tag_stack.pop())
      }
      ctx.finish_node()
    }
    _ => { ... }
  }
}
```

Key invariant: `parse_content` no longer calls `tag_stack.pop()` itself. The frame-level unwind handles it. `finish_element` also does not call `tag_stack.pop()` — the caller (`parse_element`) unwinds after `finish_element` returns.
