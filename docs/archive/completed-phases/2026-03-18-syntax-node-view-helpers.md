**Goal:** Add three accessor methods to `SyntaxNode` in seam that reduce boilerplate when writing typed views and fold algebras for any language built on loom.

**Architecture:** Three new public methods on `SyntaxNode` in `seam/syntax_node.mbt`: `token_text`, `children_from`, and `nodes_and_tokens`. Each is a pure projection from data already in CstNode's children array. No new types, no new traits, no new dependencies. The methods follow seam's existing patterns (offset accumulation via CstElement traversal, optional trivia filtering).

**Tech Stack:** MoonBit, seam module (`dowdiness/seam`)

---

## Background

Typed views in `examples/lambda/src/views.mbt` (and any future language grammar) repeatedly chain SyntaxNode navigation methods. Three patterns appear across views and fold algebras:

1. **Token text extraction** — `find_token(kind).map(t => t.text()).unwrap_or("")` (4 uses in lambda views)
2. **Child slicing** — manually iterating `children()` to skip the first N (AppExprView::args)
3. **Node/token partitioning** — iterating `all_children()` to separate nodes from operator tokens (BinaryExprView::ops, fold algebra BinaryExpr case)

These patterns will multiply in any language with function calls, infix operators, pipe chains, or comma-separated lists.

/// Text of the first direct token child matching `kind`, or "" if not found.

    None => ""
  }

### Specification

```
pub fn SyntaxNode::children_from(self : SyntaxNode, start : Int) -> Array[SyntaxNode]
```

Returns interior-node children starting from index `start` (0-indexed, tokens skipped). Like `children()` but skips the first `start` node children.

- If `start <= 0`, equivalent to `children()`.
- If `start >= child_count`, returns empty array.
- Avoids allocating SyntaxNodes for skipped children.

///|
/// Interior-node children starting from index `start` (0-indexed, tokens skipped).
/// Like `children()` but skips the first `start` node children without
/// allocating SyntaxNodes for them.
pub fn SyntaxNode::children_from(
  self : SyntaxNode,
  start : Int,
) -> Array[SyntaxNode] {
  let result : Array[SyntaxNode] = []
  let mut count = 0
  for offset = self.offset, i = 0; i < self.cst.children.length(); {
    match self.cst.children[i] {
      Node(cst_child) => {
        if count >= start {
          result.push(SyntaxNode::new(cst_child, Some(self), offset))
        }
        count = count + 1
        continue offset + cst_child.text_len, i + 1
      }
      Token(token) => continue offset + token.text_len(), i + 1
    }
  }
  result
}
```

///|
/// Partition direct children into interior nodes and leaf tokens.
/// When `trivia_kind` is provided, tokens of that kind are excluded.
/// Both arrays preserve source order. Useful for infix expressions
/// where nodes are operands and tokens are operators.
pub fn SyntaxNode::nodes_and_tokens(
  self : SyntaxNode,
  trivia_kind? : RawKind? = None,
) -> (Array[SyntaxNode], Array[SyntaxToken]) {
  let nodes : Array[SyntaxNode] = []
  let tokens : Array[SyntaxToken] = []
  let mut offset = self.offset
  for elem in self.cst.children {
    match elem {
      CstElement::Node(child_cst) => {
        nodes.push(SyntaxNode::new(child_cst, Some(self), offset))
        offset = offset + child_cst.text_len
      }
      CstElement::Token(tok) => {
        let include = match trivia_kind {
          Some(tk) => tok.kind != tk
          None => true
        }
        if include {
          tokens.push(SyntaxToken::new(tok, offset))
        }
        offset = offset + tok.text_len()
      }
    }
  }
  (nodes, tokens)
}
```
