### Changes from Design Doc

1. **Kind override syntax**: `#loom.literal("IntToken")` (positional arg) instead of `#loom.literal(kind="IntToken")` (named arg). MoonBit's `@attribute` module does not expose a `KeyValue` constructor for named args in annotations.

2. **Pattern syntax**: MoonBit uses `Variant(_)` (single underscore) for unary constructor matching, not Rust-style `Variant(..)`. The `arg_count: Int` field on `VariantDecl` is used to generate the correct number of `_` placeholders.

3. **`IntegerToken` vs `IntToken`**: The generic convention `variant_name + "Token"` would give `IntegerToken`. The lambda example expects `IntToken` (raw 10). Resolved via `#loom.literal("IntToken")` override on the `Integer` variant.

## Design Decisions

- **`emit_spec.mbt`**: Kept as 57-byte stub. The originally-planned `syntax_kind_to_token_kind` and `cst_token_matches` functions were removed from the framework in 2026-03/04 cleanups. No current framework consumer requires these. Generate only when needed.

- **RawKind numbering**: Fork (i) ADR — append-only. Seeding from existing `syntax_kind.mbt` via `--seed` ensures idempotent regeneration. Without seeding, new token kinds get sequential numbers.

- **SyntaxKind.g.mbt placement**: Generated `syntax_kind.g.mbt` is token-only (no term kinds). Not dropped into the lambda example's syntax package (which has hand-written term kinds). Future Phase 2 will generate term kinds and replace the hand-written file.

- **Lambda Show for λ**: Uses literal `λ` character in annotation (`#loom.punct("λ")`) rather than `\u03BB` to avoid parser string-escape round-trip issues.
